## 유저 여정 흐름도

1. 접속
   1. 앱 실행 및 인증
      - 유저가 `에코퀘스트` 접속
      - `Supabase Auth` 기반 로그인

   2. 유저 프로필 및 일일 퀘스트 로드
      - 유저의 기본 정보와 가상 테라리움 정보 가져오기
      - 퀘스트 확인 (일일퀘스트 만료 시 날씨 등을 통해 일일퀘스트 선정)

   3. 메인화면 렌더링
      - 유저 정보, 퀘스트 버튼, 테라리움 보여줌

        ![](ui_01.png)

2. 생물 등록 시나리오
   1. 사진 업로드
      - 메인화면의 수집하기 버튼 클릭
      - 유저가 사진 선택 후 업로드

   2. 이미지 분석 (생물 수집)
      - 분석 전 오류
        - 이미지의 위치 정보가 없을 때 관련 안내사항 다이얼로그로 보여주고 등록 과정 종료
        - 이미지 위치 정보가 보호구역일 때 관련 사항 다이얼로그로 보여주고 등록 과정 종료

      - 이미지가 Gemini Vision API로 분석됨
      - 분석 결과를 유저에게 보여줌
        - 생물이 인식되지 않거나 생물이 아닌 경우 이를 다이얼로그로 보여주고 과정 종료
        - 생물이 보호종인 경우 관련 사항 다이얼로그로 보여주고 이미지 업로드 + 등록 과정 종료
        - 생물이 등록 가능한 경우
          - 이미 등록이 된 경우 간단한 등록 화면을 보여주고 이미지 업로드 + 퀘스트 진행도 갱신
          - 새로운 생물인 경우 화려한 등록 화면 보여주고 유저의 생물 도감에 저장 + 이미지 업로드 + 퀘스트 진행도 갱신

        ![](ui_02.png)

3. 퀘스트 확인 및 보상 시나리오
   1. 퀘스트 확인
      - 퀘스트 목록을 보여주고 진행도를 간단하게 보여줌
      - 완료된 퀘스트는 보상 버튼이 활성화됨

   2. 퀘스트 보상 수령
      - 퀘스트 보상 버튼을 누르면 보상 획득 화면을 보여주고 아이템 획득 처리 진행

        ![](ui_03.png)

4. 테라리움 수정
   1. 테라리움 수정 화면 열기
      - 테라리움 클릭하여 수정 화면 열기

   2. 테라리움 object 수정
      - 아이템 클릭해 장착 옵션 선택, 이후 아이템이 장착 표시 표시
      - 장착된 아이템을 클릭해 해제 옵션 선택, 이후 장착 표시 사라짐
      - 다른 아이템 장착 시도할 때, 같은 슬롯이면 기존 장착된 아이템의 장착 표시 사라짐

        ![](ui_04.png)

5. 미니게임 시작
   1. 미니게임 열기
      - 퀘스트 목록에 미니게임 일일퀘스트의 시작 버튼 클릭

   2. 미니게임 진행
      - 사진 공개 후 객관식 보기 선택
      - 맨 아래 작게 "해당 문제의 답이 없나요?" 텍스트 버튼 클릭

   3. 미니게임 보상 획득
      - 현재 기준의 신뢰도를 바탕으로 보상 획득
        - 추후 미니게임을 통해 검증이 완료되면 신뢰도 업데이트

        ![](ui_05.png)

## 기술 흐름도 (내부 로직 및 외부 API 활용 및 과금 조사)

1. 유저의 사진 업로드
   1. 클라이언트 전처리
      - `Pillow` 라이브러리로 큰 이미지 리사이징 및 jpeg 형식 변경
      - 이미지 EXIF에서 위치 정보와 촬영 시간 추출
        - 촬영 시간이 과도하게 차이나거나 위치 정보가 없다면 예외처리 진행

   2. 적합한 촬영 위치인지 확인
      - DB의 `ProtectedAreas`을 통해 좌표가 보호 범위 내에 있는지 확인
      - 범위 내라면 안내사항 전달 후 예외처리 진행

   3. AI를 이용한 이미지 확인
      - Gemini Vision API를 통해 어떤 생물인지 판별 후 JSON 형식으로 반환 요청
        - 동작을 위한 프롬프트 예시
          - "이 사진 속 생물종을 판별하고, 이를 JSON 형식으로 반환해줘."

        - 비용
          - 매월 1,000 유닛(이미지) 무료
          - Gemini 3.1 Flash-Lite 유료 등급 사용 시 1백만 토큰당 $$0.25

        - 여러 예와처리
          - 시간이 오래 걸리면 응답을 포기하고 예외처리
          - 생물이 인식되지 않으면 예외처리

        - 인식되면 반환된 JSON의 `id` 확인

   4. 보호종 여부 확인
      - `Dictionary` 테이블 조회 후 보호종 여부 확인
        - 보호종인 경우 유저에게 안내 문구 표시, 이후 시스템적 기능은 계속 진행

   5. 데이터 저장
      - Supabase Storage에 이미지 파일 업로드
      - 저장된 위치를 통해 `Pictures` 테이블에 데이터 저장
      - 퀘스트 조건에 해당되면 진행도 갱신

2. 퀘스트 생성
   1. 매일 일일퀘스트 생성
      - 날씨 API를 통해 일일퀘스트를 생성
      - 전일 일일퀘스트 삭제

   2. 연구자 기반 퀘스트 생성
      - 연구자가 원하는 데이터(참새 데이터)의 퀘스트("참새 세 마리를 촬영해보세요!") 추가

3. 미니게임 기반 이미지 검증 프로세스
   1. 사진 선정
      - `Pictures`에서 다음 조건의 사진 선정
        - 최종 확정된 생물이 없는 사진
        - 본인이 올리지 않은 사진
        - 현재 유저가 투표하지 않은 사진

      - 사진의 보기 생성
        - AI의 답과 현재의 여러 후보들을 선정

   2. 유저 응답
      - 캡차 형식의 게임 시작

   3. 사전 확인
      - 너무 빠른 응답을 걸러냄
        - 너무 빠른 시간 내(예시: 500ms 미만) 응답 시 매크로로 간주하고 응답에서 제외

   4. 응답 기록
      - 응답을 `PictureTrust`에 저장
      - 응답이 합의 임계치를 넘으면 사진의 생물 결과를 확정
        - 예시: 5표 이상이거나 2표 이상과 80% 이상 득표율

   5. 응답 후 처리
      - 응답한 유저에게 보상 제공
      - 응답한 사진이 확정된 경우, 응답 로그를 통해 신뢰도 점수 갱신

4. 생태학자(+기관)에게 데이터 제공
   1. 데이터 요청
      - 다양한 경로로 데이터 요청을 받음

   2. 원하는 데이터를 가져오는 쿼리 작성
      - 예시: 2026년의 참새 발견 데이터

   3. 데이터 품질 필터링
      - 요청 사항에 따른 품질 필터링 설계
        - 고품질 데이터가 필요한 경우: 검증이 완료된 사진 데이터만을 사용
        - 많은 데이터가 필요한 경우: AI 추정 데이터 포함, 경향성(뜬금없는 데이터: 원주에서 바다표범 발견)이 없는 데이터 제외

   4. 데이터를 제공
      - 데이터를 연구자가 원하는 포맷으로 가공
        - 지도에 매핑하거나 사진 원본 포함 등

## DB 설계

![](DB_구조.png)

### 테이블 설명

#### 유저 및 상태 관리

- **Users**

  | **필드명**    | **설명**                                  |
  | ------------- | ----------------------------------------- |
  | `id`          | 유저 고유 ID (Supabase Auth 연동)         |
  | `trust_score` | 미니게임 응답에 따라 변동되는 신뢰도 점수 |
  | `created_at`  | 계정 생성 일시                            |
  | `xp`          | 유저의 경험치 수치                        |

- **UserInventory**

  | **필드명**    | **설명**                  |
  | ------------- | ------------------------- |
  | `id`          | 인벤토리 개별 고유 번호   |
  | `user`        | 인벤토리의 주인 (유저 ID) |
  | `item`        | 획득한 아이템 마스터 ID   |
  | `acquired_at` | 아이템을 획득한 일시      |

- **UserTerrarium**

  | **필드명**  | **설명**                       |
  | ----------- | ------------------------------ |
  | `slot`      | 장착된 테라리움 슬롯 ID        |
  | `inventory` | 슬롯에 장착된 인벤토리 객체 ID |
  | `user`      | 테라리움을 소유한 유저 ID      |

#### 기준 정보

- **ItemCategories**

  | **필드명**    | **설명**                     |
  | ------------- | ---------------------------- |
  | `id`          | 아이템 분류 고유 번호        |
  | `name`        | 분류 명칭 (예: '돌', '배경') |
  | `description` | 카테고리에 대한 부가 설명    |

- **Items**

  | **필드명**    | **설명**                                            |
  | ------------- | --------------------------------------------------- |
  | `id`          | 아이템 고유 번호                                    |
  | `name`        | 아이템 명칭                                         |
  | `expiry_date` | 시즌제 아이템의 만료 일시 (NULL일 경우 영구 아이템) |
  | `category`    | 아이템 분류 ID                                      |

- **DictionaryCategories**

  | **필드명**    | **설명**                    |
  | ------------- | --------------------------- |
  | `id`          | 생물 분류 고유 번호         |
  | `name`        | 분류 명칭 ('새', '나무' 등) |
  | `description` | 분류 설명                   |

- **Dictionary**

  | **필드명**     | **설명**            |
  | -------------- | ------------------- |
  | `id`           | 생물 도감 고유 번호 |
  | `name`         | 생물의 명칭         |
  | `description`  | 생물의 설명         |
  | `is_protected` | 법정 보호종 여부    |
  | `category`     | 소속 생물 분류 ID   |

- **TerrariumSlot**

  | **필드명**    | **설명**                                  |
  | ------------- | ----------------------------------------- |
  | `id`          | 슬롯 고유 번호                            |
  | `name`        | 슬롯 표현 이름 (예: '돌')                 |
  | `description` | 슬롯 설명                                 |
  | `category`    | 해당 슬롯에 적용(장착) 가능한 카테고리 ID |

- **ProtectedAreas**

  | **필드명**    | **설명**           |
  | ------------- | ------------------ |
  | `id`          | 보호구역 고유 번호 |
  | `name`        | 보호구역 명칭      |
  | `center_lat`  | 중심점 위도        |
  | `center_long` | 중심점 경도        |
  | `radius`      | 업로드 제한 반경   |

- **Quest**

  | **필드명**       | **설명**                                     |
  | ---------------- | -------------------------------------------- |
  | `id`             | 퀘스트 마스터 고유 번호                      |
  | `target_id`      | 퀘스트 목표 생물 (NULL = 아무 생물이나 가능) |
  | `reward_xp`      | 보상으로 주어지는 경험치                     |
  | `reward_item_id` | 보상으로 주어지는 아이템 ID                  |

- **TargetDictionary**

  | **필드명**   | **설명**                          |
  | ------------ | --------------------------------- |
  | `quest`      | 연결된 퀘스트 ID                  |
  | `dictionary` | 퀘스트의 목표 생물 도감 ID        |
  | `count`      | 해당 생물 달성에 필요한 수행 횟수 |

- **TargetDictionaryCategories**

  | **필드명** | **설명**                               |
  | ---------- | -------------------------------------- |
  | `quest`    | 연결된 퀘스트 ID                       |
  | `category` | 퀘스트의 목표 생물 분류 ID             |
  | `count`    | 해당 생물 분류 달성에 필요한 수행 횟수 |

#### 핵심 도메인 로직

- **Pictures**

  | **필드명**                | **설명**                                     |
  | ------------------------- | -------------------------------------------- |
  | `id`                      | 사진 고유 번호                               |
  | `user`                    | 사진을 업로드한 유저 ID                      |
  | `storage_url`             | Supabase Storage 이미지 경로                 |
  | `latitude`                | 촬영 위도 (연구자용 원본)                    |
  | `longitude`               | 촬영 경도 (연구자용 원본)                    |
  | `candidate_dictionary_id` | AI가 1차로 추정한 생물 도감 고유 번호        |
  | `confirmed_dictionary_id` | 미니게임 다수결을 통해 최종 확정된 생물종 ID |

- **PictureTrust**

  | **필드명**              | **설명**                                 |
  | ----------------------- | ---------------------------------------- |
  | `user`                  | 검증 미니게임에 참여한 유저 ID           |
  | `picture`               | 검증 대상 사진 ID                        |
  | `selected_candidate_id` | 유저가 정답으로 선택한 후보 종 ID        |
  | `response_time`         | 응답 소요 시간 (매크로 및 어뷰징 방지용) |

- **UserQuest**

  | **필드명**       | **설명**                                                 |
  | ---------------- | -------------------------------------------------------- |
  | `quest`          | 수행 중인 퀘스트 마스터 ID                               |
  | `user`           | 퀘스트를 수행하는 유저 ID                                |
  | `progress_count` | 목표 생물 촬영 성공 누적 횟수                            |
  | `status`         | 진행 상태 ('in_progress', 'completed', 'claimed' 중 1개) |
  | `assigned_at`    | 퀘스트 발급 일시 (일일 퀘스트 만료/갱신 검증용)          |
